# 介绍

`Vue Rex` 是一个专为 Vue 3 设计的高度灵活、类型安全且插件化的请求 Hook 库。

它的核心理念是 **“约定优于配置”** 与 **“架构解耦”**。无论你使用的是 Axios、Fetch 还是 GraphQL，只需要遵循简单的 `[data, error]` 返回约定，即可享受到自动状态管理、缓存、分页、插件扩展等强大功能。

## 核心特性

*   **⚡ 响应式状态**：自动管理 `data`、`error`、`loading` 等状态，无需手动编写冗余逻辑。
*   **🔗 架构解耦**：核心逻辑不依赖于任何具体的请求库（如 Axios）。通过 `transform` 适配器，你可以轻松接入任何异步服务。
*   **🔌 插件化架构**：支持全局或局部的插件扩展，内置了轮询、缓存、错误重试、防抖节流、窗口聚焦刷新等丰富插件。
*   **📊 强大的分页**：支持传统分页、无限滚动（追加模式），并提供自动加载更多和滚动监听功能。
*   **🛡️ 类型安全**：基于 TypeScript 编写，提供完美的类型推导，甚至包括格式化后的数据类型。
*   **💾 智能缓存**：内置内存缓存策略，支持 SWR（Stale-While-Revalidate），提升用户体验并减轻服务器压力。

## 为什么选择 Vue Rex？

在复杂的业务开发中，我们经常面临多个后端服务返回结构不一致、分页逻辑重复、请求状态管理混乱等问题。

`Vue Rex` 并不是要取代 Axios 或 Fetch，而是通过一层轻量的 Hook 封装，为你的 API 调用建立一套 **标准化的交互协议**。它让你的代码更具可读性、可维护性，并且让开发者专注于业务逻辑而非请求的细节处理。
